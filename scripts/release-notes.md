### Added
- **Cross-device sync for flashcards, RSS, and podcasts** — Your review state, RSS subscriptions and read history, and podcast subscriptions and playback position now replicate across devices. Review flashcards on your desktop and pick up exactly where you left off on your phone (or any other device), with correct due dates, reps, lapses, and review history. Read an RSS article on one device and it's marked read on the others; subscribe to a feed or podcast and it appears everywhere. Sync runs encrypted end-to-end and works even when devices are offline — a write on one device is delivered to another when it next connects.

### Fixed & Improved
- **Review history merges correctly across devices** — Reviewing the same card on two devices no longer double-counts or drops a review. Each review event is keyed by a deterministic id (`sha1(cardId | timestamp | deviceId)`), so the same event arriving twice (e.g. on reconnect) collapses to one entry, while two genuinely distinct reviews both count. SRS state (due date, stability, difficulty, interval) resolves by last-writer-wins on a monotonic sync clock.
- **RSS no longer re-surfaces read articles after reinstall** — Read/starred state previously lived only in per-device SQLite with no replication, so a fresh install lost all of it. Article read and queued state now carries per-field transition clocks (`read_at`, `queued_at`) so concurrent toggles on two devices resolve deterministically, and subscriptions propagate (and unsubscribe) via tombstones so a removed feed doesn't reappear.
- **Podcast position and played state sync** — Playback position, played status, and subscriptions now travel between devices. Position updates are debounced per-episode to avoid bloating the sync document, and each device downloads the audio itself (honoring its own Wi-Fi/storage settings) rather than replicating large audio bytes — download *intent* syncs, so a download requested on one device is honored on the others.
- **Async sync reliability (relay-side)** — The sync relay now durably stores recent encrypted sync frames, so a device that joins a room after the writer went offline receives the backlog instead of finding an empty room. This is the foundation that makes "review on desktop now, open the phone an hour later" actually work in practice.
- **Custom YouTube transcript server** — A self-hosted transcript API server can now be configured as an alternative transcript source, with a dedicated integration tab and credentials stored in settings.
