# Incrementum ユーザーハンドブック

**インクリメンタル・リーディングと間隔反復を使いこなすための完全ガイド**

---

## はじめに

### インクリメンタムとは何ですか?

Incrementum は、実証済みの 2 つの技術を組み合わせた強力な学習アプリケーションです。

**インクリメンタル・リーディング** - 大量の情報を、時間をかけて管理しやすい小さな単位に分けて処理します。記事を最初から最後まで読むのではなく、重要なポイントを抽出し、徐々に理解を深めていきます。

**間隔反復** - 科学的に最適化された間隔で教材を復習し、記憶の定着を最大化します。FSRS-5 や SM-18 などのアルゴリズムは、忘れそうになる時期を予測し、ちょうどよいタイミングで復習をスケジュールします。

### 重要な概念

- **ドキュメント**: ソース資料 (PDF、EPUB、記事、ビデオ)
- **抜粋**: 文書から抽出した重要なポイントまたはセクション
- **学習アイテム**: 抜粋から作成されたフラッシュカードまたは Q&A カード
- **キュー**: レビューが予定されている項目。優先度別に整理されています。
- **レビュー**: 知識を積極的に思い出して評価するセッション

## はじめに

### 初回起動

Incrementum を初めて起動すると、次の 4 つの主要セクションを含む **ダッシュボード** が表示されます。

1. **キュー** - レビューキュー（最初は空）
2. **レビュー** - アクティブなレビュー セッション
3. **ドキュメント** - ドキュメント ライブラリ
4. **分析** - 進捗状況の統計

### 初期セットアップ

1. **テーマの選択** - [設定] → [外観] → [テーマ] に移動します。
   - 17 個の組み込みテーマが利用可能 (6 個のダーク、11 個のライト)
   - モダンな外観には「モダン ダーク」または「マテリアル ユー」をお試しください

2. **レビュー設定を構成する** - 設定 → 学習 → アルゴリズム
   - **アルゴリズム**: FSRS-5 (推奨)、SM-18、または SM-2
   - **希望の記憶力**: 90% (デフォルト) - どの程度記憶に留めたいかを目標とします。
   - **1 日あたりの学習**: 初心者に推奨される 20 ～ 50 項目

3. **カテゴリの設定** - [設定] → [カテゴリ]
   - さまざまな主題のカテゴリを作成します (例: 「プログラミング」、「科学」、「言語」)
   - カテゴリは学習教材の整理とフィルタリングに役立ちます

### 最初のドキュメント

最初のドキュメントをインポートしましょう。

1. サイドバーの **ドキュメント** をクリックします
2. [**インポート**] ボタン (右上) をクリックします。
3. インポート方法を選択します。
   - **ローカル ファイル**: PDF、EPUB、またはテキスト ファイルを選択します
   - **URL**: 任意の Web URL を貼り付けます
   - **Arxiv**: 研究論文 ID または URL を貼り付けます
4. 処理を待ちます (自動セグメンテーションが自動的に開始されます)。

## 文書管理

### インポート形式

|フォーマット |説明 |使用例 |
|----------|---------------|----------|
| **PDF** |ポータブルドキュメントフォーマット |研究論文、電子ブック、ドキュメント |
| **EPUB** |電子出版 |リフロー可能なテキストを含む書籍、記事 |
| **マークダウン** | `.md` ファイル |技術文書、メモ |
| **HTML** |ウェブページ |記事、ブログ投稿 |
| **Anki (.apkg)** | Anki デッキパッケージ | Anki からの移行 |
| **スーパーメモ** | ZIP エクスポート | SuperMemo からの移行 |
| **URL** |任意の Web リンク |オンライン記事、ブログ |
| **Arxiv** |学術論文 |研究文献 |
| **スクリーンショット** |スクリーンキャプチャ |あらゆるアプリからのクイック キャプチャ |

### ドキュメントのインポート

#### 方法 1: ローカル ファイル

1. [**ドキュメント**] → [**インポート**] をクリックします。
2. **ローカル ファイル** を選択します
3. ファイルに移動して選択します
4. Incrementum は次のことを行います。
   - テキストコンテンツの抽出
   - セクションへの自動分割
   - 読書時間と単語数を計算します
   - メタデータ（タイトル、著者など）を抽出します。

#### 方法 2: URL インポート

1. 任意の Web URL をコピーします
2. [**ドキュメント**] → [**インポート**] → [**URL**] をクリックします。
3. URLを貼り付けます
4. [**インポート**] をクリックします。
5. Incrementum がコンテンツを取得して処理します

**サポートされているサイト:**
- ニュース記事 (ほとんどの主要サイト)
- ブログ投稿
- ドキュメントページ
- ミディアム、サブスタックなど

#### 方法 3: Arxiv 論文

1. Arxiv の論文を見つけます (例: `https://arxiv.org/abs/2301.07041`)
2. URL または用紙 ID (`2301.07041`) をコピーします。
3. **ドキュメント** → **インポート** → **Arxiv** をクリックします
4. URLまたはIDを貼り付けます
5. Incrementum は以下をダウンロードします。
   - 完全なPDF
   - 要約
   - 著者
   - 発行日
   - 参考文献

### ドキュメントビューア

インポートしたら、任意のドキュメントを開いてアクセスします。

**ビューアの機能:**
- **ページ ナビゲーション**: ページ/セクションをスクロールします。
- **ズーム**: テキストのサイズを調整します
- **全画面**: 気を散らすことなく読書が可能
- **検索**: ドキュメント内のテキストを検索します
- **目次**: セクションにジャンプします (可能な場合)

**注釈ツール:**
1. **テキストのハイライト**: テキストを選択 → ハイライトの色を選択
   - 黄色: 重要な概念
   - 緑: 例
   - 青: 定義
   - 赤: クリティカルポイント
   - 紫: 関連トピック

2. **抽出の作成**: テキストを選択 → 「抽出」ボタンをクリック
   - 抽出が「抽出」タブに表示されます
   - 後でフラッシュカードに変換できます

3. **メモを追加**: テキストを選択 → 「メモ」ボタンをクリックします
   - あなたの考え/メモを添付してください
   - メモは抜粋とともに表示されます

### 文書の構成

**カテゴリ:**
- 各ドキュメントをカテゴリに割り当てます
- カテゴリごとにドキュメントをフィルタリングします
- カテゴリは抽出とカードに継承されます

**タグ:**
- ドキュメントにカスタムタグを追加する
- タグを使用してカテゴリを横断して整理する
- 例: `#緊急`、`#リサーチ`、`#チュートリアル`

**検索:**
- すべてのドキュメントにわたる全文検索
- カテゴリ、タグ、日付範囲によるフィルタリング
- タイトル、日付、文字数で並べ替え

## 学習システム

### FSRS-5 について理解する

**FSRS-5** (Free Spaced Repetition Scheduler) は、次のような最新のアルゴリズムです。

1. **メモリ状態を追跡**: 各カードのメモリ強度をモデル化します。
2. **忘れ物を予測**: 各アイテムを忘れる時期を予測します
3. **スケジュールの最適化**: 最適な時間にレビューをスケジュールします。
4. **あなたに適応**: あなたのパフォーマンスパターンから学習します

**主要な指標:**
- **安定性**: 記憶がどのくらい持続するか (高い = より安定)
- **難易度**: そのアイテムがあなたにとってどれくらい難しいか (1 ～ 10 スケール)
- **想起確率**: 現在思い出せる確率 (0 ～ 100%)

### 評価システム

レビュー中に、思い出した内容に基づいて各項目を評価します。

|評価 |ラベル |説明 |一般的な間隔 |
|----------|------|---------------|------|
| **1** |もう一度 |完全停電 | ～10分 |
| **2** |ハード |多大な労力をかけて思い出した | 1～2日 |
| **3** |良い |少し考えて思い出した | 5～7日 |
| **4** |簡単 |リコールは簡単でした | 10-14日 |

**プレビュー間隔:**
評価する前に、Incrementum は 4 つの評価オプションすべてについて、各カードが次にいつ表示されるかを正確に表示します。スケジュールを最適化するためにこれを使用してください。

### カードの種類

#### 1. 基本的なフラッシュカード
シンプルな表裏カード

**表面:** フランスの首都はどこですか?
**裏面:** パリ

**以下に最適:** 事実、定義、語彙

#### 2. 穴埋め（Cloze）
穴埋め形式

**本文:** {{フランス}}の首都はパリです。

**次のように表示されます:** _____ の首都はパリです。

**最適な用途:** 文脈の中での学習、概念どうしの関係理解

#### 3. Q&A カード
質問と回答のペア

**Q:** TCP と UDP の違いを説明してください。
**A:** TCP は接続指向であり、配信が保証されています。 UDP はコネクションレスであり、保証はありません。

**こんな方に最適:** 概念、説明

#### 4. 画像のオクルージョン
画像の一部を非表示にする (図、チャート)

**用途:** 解剖学、地図、図表

### カードの作成

#### 抜粋より

1. 読書中に重要なテキストを選択します
2. [**抽出**] をクリックして抽出を作成します
3. [**抽出**] タブで、抽出を確認します。
4. 任意の抽出で [**カードの作成**] をクリックします。
5. カードの種類を選択します (フラッシュカード、Cloze、Q&A)
6. カードの内容を編集する
7. [**保存**] をクリックします。

カードの審査が予定されています。

#### マニュアルの作成

1. [**キュー**] → [**アイテムの追加**] をクリックします。
2. カードの種類を選択します
3. 前後のコンテンツを入力します
4. カテゴリを選択します
5. [**作成**] をクリックします。

#### AI を活用した発電

AI が構成されている場合:

1. 抽出またはドキュメントのセクションを選択します
2. [**カードの生成**] をクリックします。
3. AIが複数のカードを自動作成
4. 必要に応じて確認および編集します
5. 最良のものを保存する

## レビュープロセス

### レビューセッションの開始

1. サイドバーの **レビュー** をクリックします
2. 今日（および今後の）締め切りのカードを確認する
3. [**レビューの開始**] をクリックして開始します。

### レビューインターフェイス

**カード表示:**
- カードの表面を表示 (質問またはプロンプト)
- **スペース**を押すかクリックして答えを表示します
- 答えは下に表示されます

**混合レビュー セッション (カード + ドキュメント):**
- レビュー セッションには、読む予定の **学習項目** や **ドキュメント**を含めることができます。
- ドキュメントが表示されたら、セッション カードから直接開くことができます。
- カードが次のレビューをスケジュールするのと同じように、ドキュメントを評価すると、次の閲覧日がスケジュールされます。

**評価インターフェイス:**
回答を明らかにすると、4 つの評価ボタンが表示されます。

「」
[もう一度] [難しい] [良い] [簡単]
  ～10分 ～2日 ～7日 ～14日
「」

各ボタンには **次のレビュー日** が表示されます。これは **プレビュー間隔** 機能です。

**回復アクション (レビュー キュー インスペクター):**
学習項目のスケジュールを簡単に変更する必要がある場合は、これらを使用します。

- **間隔を圧縮**: 次のレビューを近づけます (間隔を短くします)。
- **インテリジェントにスケジュールを変更**: アイテムを「今すぐ期限」に移動します。
- **ダウングレードの頻度**: 次のレビューを延期します (間隔が長くなります)。

これらのアクションは**学習項目のみ**に適用され、スケジュールはすぐに更新されます。

### キーボード ショートカット (レビュー モード)

|キー |アクション |
|-----|----------|
| 「スペース」 |答えを表示 |
| `1` | 「もう一度」評価 |
| `2` |評価「ハード」 |
| `3` |評価「良い」 (推奨デフォルト) |
| `4` | 「簡単」と評価 |
| `Ctrl+Enter` |答えを表示 |
| `Ctrl+1/2/3/4` |回答を表示せずに評価する |
| `Esc` |セッションを一時停止/終了する |
| `Ctrl+E` |現在のカードを編集 (未実装) |
| `Ctrl+D` |現在のカードを削除 (グローバルでも「ダッシュボードに移動」に使用) |

### セッション管理

**レビュー セッションの機能:**
- **進行状況バー**: 残りのカードを表示します
- **時間追跡**: セッションの継続時間を表示します
- **ブレイクタイマー**: N 枚ごとにオプションでブレイクします
- **セッション制限**: セッションあたりの最大カードまたは時間を設定します

**セッションの終了:**
- 完了したら [**完了**] をクリックします
- または制限を設定します (設定 → レビュー → セッション制限)
- 未完成のカードは次のセッションまで残っています

### 戦略を検討する

#### 毎日のレビュールーチン

1. **午前のセッション** (15 ～ 30 分)
   - 今夜締め切りのカードを確認する
   - より難しいアイテムに焦点を当てる

2. **イブニングセッション** (15-30分)
   - 日中に追加されたレビューカード
   - 読み取りから新しいカードを作成する

#### バックログの管理

期限切れのカードが多数ある場合 (100 枚以上):

1. **新しいカードに重点を置く**: レビューを 1 日あたり 20 ～ 30 件に制限する
2. **フィルターを使用**: カテゴリ別にレビューします (やりすぎないでください)
3. **学習セッション**: 週末の追いつきセッション
4. **維持率を調整**: 一時的に 85% に低下します (レビューが少なくなります)

#### 「また」カードの扱い

「再び」と評価されたカードはすぐに再表示されます (10 分)。戦略:

- **即時再学習**: 同じセッション内でカードを再確認します
- **別セッション**: 後日カードをもう一度確認します
- **問題について**: 再度多い場合は、カードの書き込みが不十分である可能性があります

## キュー管理

### キューを理解する

**キュー**には、レビューが予定されているすべてのアイテムが含まれており、次のように整理されています。

- **期限**: 期限が早いアイテムが最初に表示されます
- **優先度**: ユーザーが設定した優先度 (0-100)
- **カテゴリ**: 主題分野
- **カードの種類**: フラッシュカード、クローゼなど。

### キュービュー

#### 期限付きビュー
今日期限のアイテムと期限を過ぎたアイテムを期限順に並べて表示します

#### スケジュールされたビュー
今後のレビューを含む、スケジュールされたすべてのアイテムを表示します

#### 新しいビュー
新しく作成されたまだレビューされていないカードを表示します

### キュー操作

**フィルタリング:**
- カテゴリ別 (例: 「プログラミングのみを表示」)
- カードの種類別 (例: 「Cloze カードのみを表示」)
- 優先度範囲別 (例: 「優先度 80+ を表示」)

**並べ替え:**
- 期日 (デフォルト)
- 優先順位
- 難易度
- ランダム（多様性のため）

**一括アクション:**
1. 複数の項目（チェックボックス）を選択します
2. アクションを選択します。
   - **カテゴリの変更**: 別のカテゴリに移動します
   - **優先度の設定**: 一括更新の優先度
   - **一時停止**: レビューを一時的に非表示にします
   - **削除**: 完全に削除します

### 優先システム

任意の項目に優先度 0 ～ 100 を設定します。

- **100**: クリティカル (学習する必要があります)
- **80-90**: 重要
- **60-70**: 通常の優先度
- **40-50**: 優先度が低い
- **0-20**: アーカイブ/参照

**優先スケジュール:**
優先度の高い項目は、混合レビューでより頻繁に表示されます。

### スマートキュー

フィルターを使用してカスタム キューを作成します。

**キューの例:**
- 「今日のフォーカス」: メインカテゴリーのカードが期限付き
- 「クイックレビュー」: 簡単なカード、優先度 < 50
- 「Deep Dive」: 研究カテゴリーのハードカード
- 「試験準備」: 「生物学」カテゴリのすべてのカード

**スマート キューの作成:**
1. [**キュー**] → [**保存されたキュー**] をクリックします。
2. [**新しいキュー**] をクリックします。
3. フィルターと並べ替え順序を設定する
4. 名前を付けて保存します

## 分析と進捗状況の追跡

### ダッシュボードの概要

Analytics ダッシュボードは、包括的な洞察を提供します。

**主要な指標:**
- **今日期限のカード**: 審査待ちの番号
- **合計カード数**: システム内のすべてのカード
- **定着率**: 記憶されている割合
- **連続学習**: 連続したアクティビティの日数
- **学習したカード数**: 作成されたカードの総数

### アクティビティ チャート

**30 日間のアクティビティ:**
- 1 日あたりのレビューを示す棒グラフ
- 評価ごとに色分け (再度/難しい/良い/簡単)
- 学習習慣のパターンを特定する

**学習曲線:**
- カードの合計を時系列で示す折れ線グラフ
- 知識ベースの成長を追跡する

### 統計

**統計のレビュー:**
- 完了した合計レビュー
- 平均評価分布
- 日/週/月ごとのレビュー

**カード統計:**
- カテゴリ別の合計カード
- タイプ別のカード (フラッシュカード、クローズなど)
- 新しいカードと成熟したカードの比較

**アルゴリズム メトリクス (FSRS/SM-18):**
- 平均的な安定性
- 平均的な難易度
- 予測された保持率
- メモリ性能

### カテゴリの内訳

サブジェクト領域ごとにパフォーマンスを表示します。

- カテゴリごとのカード
- カテゴリごとの維持率
- カテゴリごとのアクティビティレベル
- 得意分野と苦手分野を特定する

### ゴールと連続記録

**目標の設定:**
1. [**分析**] → [**目標**] をクリックします。
2. 日次/週次の目標を設定します。
   - 確認するカード
   - 作成するカード
   - 勉強時間
3. 進捗状況の視覚的なインジケーターを追跡する

**連続学習:**
- アクティビティが続く日
- 現在のストリークがダッシュボードに表示されます
- モチベーションを維持するために連続記録を維持する

### 統計のエクスポート

分析のためにデータをエクスポートします。

### 学習設定

#### アルゴリズムの選択

Incrementum は 3 つのスケジューリング アルゴリズムをサポートしています。あなたの学習スタイルに最適なものを選択してください。

**FSRS-5 (推奨):**
- 研究に裏付けられた最新の
- 個人の記憶に適応します
- 忘れる時間を予測します
- レビューが少なくなり、定着率が向上

**SM-18 (スーパーメモ 18):**
- オリジナルのアプリケーションからリバースエンジニアリングされた最新の SuperMemo アルゴリズム
- 難易度、安定性、取得可能性にわたる 3D SInc (安定性増加) ルックアップ マトリックスを使用します。
- トレーリング平均更新による明示的な難易度追跡
- 指数忘却曲線モデル: `R = 0.9^(t/S)`
- 経過に依存した安定性の低下による高度な障害処理

**SM-2 (クラシック):**
- 従来の SuperMemo 2 アルゴリズム (公的文書化)
- よりシンプルで予測可能
- さらなるレビューが必要

#### パラメータ

**望ましい保持力:** 0.70 - 0.95
- **90%** (デフォルト): 保持とレビュー負荷のバランスをとります。
- **85%**: レビューが少なく、維持率もわずかに低下
- **95%**: 最大の保持率、より多くのレビュー

**1日あたりの学習量:** 10 - 100
- **20** (デフォルト): ほとんどのユーザーが管理可能
- **50**: 集中的な学習期間用
- **10**: レビュー負荷が軽い

**1 日あたりのレビュー:** 50 - 500
- **200** (デフォルト): 妥当な 1 日あたりの制限
- **500**: バックログのクリア用
- **50**: 軽いレビュー日

#### インターバル設定

**新しいカードの間隔:**
- 卒業間隔 (良い評価): 1 ～ 10 日
- 簡単な間隔: 3 ～ 21 日
- 最小間隔: 1 日

**最大間隔:**
- 最長間隔の上限 (デフォルトは 365 日)
- カードがあまりにも大幅にスケジュールされるのを防ぎます

**長尺安全キャップ (ビデオ/記事):**
- 長いビデオ/記事の場合、肯定的な評価 (「良い」/「簡単」) はカバレッジを考慮しています。
- 推定コンテンツ時間の **25%** に満たない場合、次の間隔は **1 日**に制限されます。
- 支出が **50%** 未満の場合、次の間隔は **2 日**に制限されます。
- 支出が **75%** 未満の場合、次の間隔は **4 日**に制限されます。
- これにより、部分的な進行後に長い形式のコンテンツが大幅にスケジュールされるのを防ぎます。
- 適用されると、スケジューラの理由には、透明性を確保するための **期間を考慮した上限** の注記が含まれます。

#### バックアップと復元

Incrementum は、学習データを保護し、デバイス間で移行するための完全なバックアップおよび復元システムを提供します。

#### 完全なアプリのバックアップ

**バックアップされる内容:**
- **設定**: すべての設定、テーマ、学習パラメータ
- **ドキュメント**: メタデータを含むすべてのインポートされたドキュメント
- **抜粋**: すべてのハイライトと抽出されたコンテンツ
- **学習アイテム**: すべてのフラッシュカード、クローズの削除、Q&A カード
- **スケジュール データ**: アルゴリズムのメモリ状態 (安定性、難易度、間隔)、期日
- **コレクション**: すべてのコレクションとドキュメントの割り当て
- **UI 状態**: サイドバーの状態、テーマの設定
- **オプション**: 実際のドキュメント ファイル (PDF、EPUB など)

**バックアップの作成:**

1. **[設定] → [インポート/エクスポート] → [アプリの完全バックアップ]** に移動します。
2. [**バックアップと復元を開く**] をクリックします。
3. **バックアップのエクスポート** を選択します
4. オプションのラベルを追加します（例：「PC を再フォーマットする前」）
5. ドキュメント ファイルを含めるかどうかを選択します。
   - **メタデータのみ**: ファイルが小さい (~KB-MB)、ファイルを個別に再インポートします
   - **インクルード ファイル**: より大きなファイル (~MB ～ GB)、完全な自己完結型バックアップ
6. **[バックアップのエクスポート**] をクリックし、`.incrementum` ファイルを保存します。

**ファイル形式:**
- 拡張子: `.incrementum`
- 形式: ヘッダーコメント付きの JSON
- 命名: `incrementum-backup-[ラベル]-[日付]-[時間].incrementum`

**バックアップからの復元:**

1. **[設定] → [インポート/エクスポート] → [アプリの完全バックアップ]** に移動します。
2. [**バックアップと復元を開く**] をクリックします。
3. **バックアップのインポート**を選択します
4. `.incrementum` ファイルを選択します
5. バックアップの内容をプレビューします。
   - 文書数
   - 抽出数
   - 学習項目数
   - コレクション数
   - ファイルが含まれるかどうか
6. インポート オプションを構成します (オプション)。
   - **何をインポートするか**: 特定のデータ型を選択してください
   - **重複処理**: スキップ、置換、またはマージ
   - **ファイルのインポート**: 文書ファイルを復元するかどうか
7. [**バックアップのインポート**] をクリックします。
8. インポートが完了するまで待ちます (進行状況が表示されます)。

**重複処理戦略:**
- **スキップ**: すでに存在する項目をスキップします (ほとんどの場合に推奨)
- **置換**: 既存のアイテムをバックアップ バージョンで上書きします
- **マージ**: すべてのアイテムの新しいコピーを作成します (重複が作成される可能性があります)

**使用例:**

|シナリオ |推奨されるアプローチ |
|----------|----------|
| **新しいコンピュータに移行** |ファイルとともにエクスポートし、新しいマシンにインポート |
| **大きな変更の前にバックアップ** |メタデータのみの迅速なバックアップ |
| **デバイス間で同期** |エクスポート/インポートのワークフロー |
| **コレクションを共有する** |特定のコレクションをエクスポートする |
| **古いデータをアーカイブする** |エクスポートして長期保存 |
| **再フォーマット後の復元** |ファイルを含む完全なバックアップをインポート |**重要な注意事項:**
- **スケジュールの保存**: すべてのアルゴリズム タイプのすべてのスケジュール データ (安定性、難易度、期日) が正確に保存されます。
- **ファイル パス**: ファイルなしでインポートする場合は、元のドキュメントを再インポートする必要があります。 Incrementum はコンテンツ ハッシュによってそれらを照合し、メタデータを復元します
- **バージョン互換性**: バックアップには上位互換性がありますが、古いバージョンのアプリでは動作しない可能性があります
- **ストレージ**: バックアップを安全に保管します - バックアップには個人の学習データが含まれます

  - フラッシュカード

4. Incrementum は、スレッド/ソースのメタデータを使用して NotebookLM にリンクされた抽出を作成します
5. 保存された回答には、重複を避けるために **すでに保存されました** インジケーターが表示されます

**ドキュメント Q&A + NotebookLM ワークフロー:**
1. Incrementum でドキュメントを開く

5. これらの抜粋からフラッシュカード/クローズ/Q&A アイテムを生成します。**トラブルシューティング:**
- アーティファクトのプレビューでメディアが利用できないと表示された場合は、NotebookLM の生成が完了するまで待って、アーティファクトを再度開きます。
- 「cli」プロバイダーを使用する場合は、NotebookLM サイドカー/CLI がビルドで利用可能であることを確認してください。
- プロバイダーを変更した場合、または認証が期限切れになった場合は、「統合」→「NotebookLM」で再接続します。

**ニュースレター フィードの検出:**

Incrementum は、人気のあるニュースレター プラットフォームから RSS フィードを自動的に検出できます。

## ヒントとベストプラクティス

### カードの作成

**実行:**
- カードを具体的にする (カードごとに 1 つの事実)
- シンプルで明確な言葉を使用する
- 回答にコンテキストを含める
- 関連する例を追加します
- 人間関係にはclozeを使用する
- 質問は簡潔にしましょう

**禁止事項:**
- 複数の事実を 1 枚のカードに入れる
- 曖昧な表現を使用する
- 質問が簡単すぎたり、難しすぎたりする
- 大きなテキストブロックをコピーする
- 定義のない略語を使用する

**例 - 悪いカード:**
「」
Q: ミトコンドリアの機能とは何ですか?また、どのように機能するのですか?
それは細胞呼吸におけるATP生成に関係しているのでしょうか？
A: [段落の説明]
「」

**例 - 良いカード:**
「」
カード 1:
Q: ミトコンドリアの主な機能は何ですか?
A: 細胞呼吸を通じて ATP を生成します。

カード 2:
Q: ミトコンドリアはどのようなプロセスを使用して ATP を生成しますか?
A: 細胞呼吸（好気性）

カード 3:
Q: ミトコンドリアによって生成されるエネルギー通貨とは何ですか?
A：ATP（アデノシン三リン酸）
「」

### 勉強のルーチン

**毎日のスケジュール (20 ～ 30 分):**
1. **午前**: 期日カードを確認する (15 分)
2. **一日中**: 読書からの抜粋を作成します
3. **夕方**: 抜粋からカードを作成します (10 ～ 15 分)

**週間スケジュール:**
- **月～金**: 定期的なレビューとカードの作成
- **土曜日**: 長時間の勉強会 (1 ～ 2 時間)
- **日曜日**: 分析のレビュー、目標の調整、整理

**大容量の管理:**
- 1 日のレビュー制限を設定します (例: 50 枚のカード)
- カテゴリごとに優先順位を付ける（1 つのテーマに焦点を当てる）
- スマートキューを使用してタスクを分割する
- 20～30分ごとに休憩を取る

### 保持の最適化

**定着率の向上:**
- 正直に評価してください (評価を水増ししないでください)
- 一貫してレビューします (毎日がベストです)
- 十分な睡眠をとる（睡眠中に記憶が定着します）
- アクティブリコール (覗かないで、まず考えてください)
- 間隔をあけてレビューする（詰め込みすぎない）

**物忘れへの対処:**
- 通常は 10 ～ 20% 忘れます (目標の保持率に応じて)
- 「もう一度」カードは学習の機会です
- 頻繁に忘れる場合 (>30%)、次のことを考慮してください。
  - 望ましい保持率を下げる (85 ～ 90%)
  - よりシンプルなカードの作成
  - コンテキストをさらに追加する
  - より頻繁にレビューする

### カテゴリの組織

**ベストプラクティス:**
- まず大まかに開始し、次に細分化します
- 例: `プログラミング` → `プログラミング/Python` → `プログラミング/Python/非同期`
- 一貫した命名を使用する
- あまり多く作成しないでください (5 ～ 10 個は管理可能です)
- 使用されていないカテゴリをマージします

**カテゴリ構造の例:**
「」
§── プログラミング
│ §── パイソン
│ §── 錆びる
│ └── アルゴリズム
§── 言語
│ §── スペイン語
│ └── 日本語
§── 科学
│ §── 物理学
│ └── 生物学
└── プロフェッショナル
    §── プロジェクト管理
    ━── システム設計
「」

### 優先順位の管理

**優先ガイドライン:**
- **100 (重要)**: 試験準備、緊急作業プロジェクト
- **80-90 (高)**: 現在のコース、アクティブ ラーニング
- **60-70 (中)**: 継続的な関心、一般知識
- **40-50 (低)**: 知っておくと便利な補足
- **0-20 (アーカイブ)**: 参照のみ、レビューはほとんどありません

**優先スケジュール:**
- 毎日のレビューで 80 以上の優先順位に焦点を当てる
- 数日ごとに 60 ～ 70 をレビューします
- 毎週 40 ～ 50 件のレビューを行う
- 毎月またはオンデマンドで 0 ～ 20 件をレビューします

### プレビュー間隔の使用

**プレビュー間隔** 機能は、4 つの評価すべてについて、各カードが次にいつ表示されるかを正確に表示します。

**使用方法:**
1. カードを読み取ります
2. 評価ボタンの下にあるプレビュー間隔を確認します。
3. 以下に基づいて評価を選択します。
   - 現在の思い出
   - どれくらい早くもう一度見たいですか
   - あなたのスケジュール (例: 試験が近づいています)**戦略の例:**
- 2 週間後の試験: 重要なカードを「簡単」と評価すると、すぐにもう一度表示されます
- 忙しい日: レビューの間隔を空けるために「良い」または「簡単」と評価します
- マスターしたい: 「難しい」と評価して、より頻繁に復習してください

### 圧倒される状況に対処する

**期限切れのカードが多すぎますか?**
1. レビュー制限を設定します (設定 → レビュー → 1 日あたりの最大数)
2. 優先度の高い項目に焦点を当てる
3. 優先度の低いカテゴリーを一時的に停止する
4. 望ましい保持力をわずかに下げることを検討してください

**コンテンツが多すぎて処理できませんか?**
1. ドキュメントを段階的にインポートする
2. 重要なポイントのみを抽出します（すべてではありません）
3. カードを選択して作成する
4. カテゴリを使用して整理する

**燃え尽き症候群?**
1. 休憩します (大丈夫です!)
2. 1 日あたりの制限を減らす
3. 重要でないカテゴリを一時停止する
4. 一度に 1 つのカテゴリに焦点を当てる

## トラブルシューティング

### 一般的な問題

#### カードがレビューに表示されない

**考えられる原因:**
- 今日レビューされたすべてのカード
- カードの停止
- アクティブな非表示カードをフィルタリングします

**解決策:**
1.「レビュー」タブで「期限」数を確認する
2. キューを確認 → カードが一時停止されていないことを確認します
3. フィルターをクリアする
4. レビュー日を確認します (将来予定されているカードの可能性があります)

#### 定着率が低い

**症状:** 多くのカードを忘れ、頻繁に「またか」と評価される

**解決策:**
1. **カードの品質を確認します**: カードは透明ですか?カードごとに事実は 1 つですか?
2. **望ましい保持率を下げる**: 90% ではなく 85% を試してください。
3. **より頻繁にレビューする**: 詰め込みではなく、毎日レビューします。
4. **コンテキストの追加**: 回答の詳細情報
5. **カードを簡素化**: 複雑なカードをより単純なカードに分割します

#### 同期の競合

**症状:** カードが重複し、同期後にデータが不一致になる

**解決策:**
1. 競合解決戦略を選択します ([設定] → [同期])
   - **ローカル勝利**: 変更を保存します
   - **リモート勝利**: サーバーの変更を受け入れる
   - **質問**: 各競合を手動で解決します
2. 定期的に同期して競合を最小限に抑える
3. 1 つのプライマリ デバイスを使用する

#### インポートの失敗

**症状:** ドキュメントのインポートが失敗するかエラーが発生する

**解決策:**
1. **ファイル形式を確認**: サポートされている形式 (PDF、EPUB など) を確認します。
2. **ファイル サイズを確認**: 非常に大きなファイルはタイムアウトになる可能性があります
3. **URL を確認**: 一部のサイトは自動アクセスをブロックします
4. **インターネットを確認**: URL インポートには接続が必要です
5. **代替案を試す**: Web コンテンツにはコピー＆ペーストを使用します

#### パフォーマンスの問題

**症状:** 読み込みの遅さ、遅延、フリーズ

**解決策:**
1. **大規模なデータベース**: 古いカードをアーカイブします (設定 → データ → アーカイブ)
2. **画像が多い**: 画像の読み込みが遅くなる
3. **システム リソース**: 他のアプリを閉じます
4. **データベースの再構築**: 設定 → データ → 再構築 (最後の手段)

#### OCR が機能しない

**症状:** OCR が失敗するか、悪い結果が得られる

**解決策:**
1. **API キーを確認**: 有効でクレジットがあります
2. **画質を確認**: 鮮明な高解像度の画像が最適です。
3. **言語を確認**: 正しい言語が選択されています
4. **代替プロバイダーを試す**: 特定のコンテンツではより適切に機能するプロバイダーもあります
5. **ローカル OCR**: インターネットに問題がある場合は Tesseract を使用してください

### 助けを求める

**リソース:**
- **ドキュメント**: 詳細なガイドについては、`docs/` フォルダーを確認してください。
- **GitHub の問題**: バグと機能リクエストを報告する
- **コミュニティ**: ディスカッションに参加したり、質問したりできます
- **キーボード ショートカット**: アプリ内で「?」を押すとクイック リファレンスが表示されます

**デバッグモード:**
問題のトラブルシューティングを行うには、デバッグ ログを有効にします ([設定] → [詳細設定] → [デバッグ モード])。

**データのエクスポート:**
大きな変更を加える前にデータをエクスポートします (設定 → バックアップ → エクスポート)

### 回復

**誤って削除した場合:**
1. バックアップを確認します (設定 → バックアップ)
2. 最近のバックアップから復元する
3. 利用可能なバックアップがない場合はサポートに連絡してください

**データベースが破損しています:**
1. データをすぐにエクスポートする
2. データベースを再構築します (設定 → データ → 再構築)
3. エクスポートされたデータをインポートする
4. 存在するすべてのデータを確認します

**進行状況の喪失:**
1.「分析」→「履歴データのエクスポート」を確認します。
2. 必要に応じてバックアップから復元します
3. クラウドプロバイダーと同期します（有効になっている場合）

## 用語集

**抽出**: ドキュメントから抽出されたコンテンツの一部、潜在的なカード素材

**学習項目**: 学習する任意の項目 (フラッシュカード、クイズ、Q&A など)

**キュー**: レビューが予定されているすべてのアイテム (優先度別に整理)

**レビュー セッション**: カードを積極的に思い出して評価する期間

**FSRS**: Free Spaced Repetition Scheduler、レビュー タイミングを最適化する最新のアルゴリズム (FSRS-5 が現在のバージョン)

**間隔**: レビュー間の時間 (例: 7 日)

**安定性**: メモリの持続時間 (FSRS メトリクス)

**難易度**: アイテムがあなたにとってどの程度難しいかを 1 ～ 10 のスケールで示します (FSRS メトリック)

**検索可能性**: 現在の再現確率、0 ～ 100% (FSRS メトリック)

**望ましい定着率**: 目標定着率 (通常 90%)

**プレビュー間隔**: 各評価オプションの次回レビュー日を表示する機能

**Cloze**: 穴埋めカードのタイプ

**一時停止**: レビューからアイテムを一時的に非表示にします

**カテゴリ**: 組織の対象領域

**タグ**: クロスカテゴリー組織用のカスタムラベル

**優先度**: ユーザーが設定した重要度 (0 ～ 100)

## よくある質問

**Q: Incrementum にニュースレターを追加するにはどうすればよいですか?**
A: ニュースレターは 2 つの方法で追加できます。
1. **ニュースレター ディレクトリ**: RSS → ニュースレター アイコン (📬) → 厳選されたニュースレターを参照して購読する
2. **ダイレクト URL**: ニュースレターの URL (Substack、Beehiiv など) → RSS → フィードの追加 → URL を貼り付けます。 Incrementum は RSS フィードを自動検出します。

**Q: どのニュースレター プラットフォームがサポートされていますか?**
A: Incrementum は、Substack、Beehiiv、Ghost ブログ、Buttondown、ConvertKit、Revue、Medium、WordPress サイトからの RSS フィードをサポートしています。ほとんどのニュースレターは RSS フィードを公開しています。ニュースレターの Web サイトで RSS リンクを確認するか、URL に「/feed」を追加してみてください。

**Q: 1 日に何枚のカードを確認する必要がありますか?**
A: 1 日あたり 20 ～ 50 個から始めてください。スケジュールや目標に基づいて調整してください。一貫性は量よりも重要です。

**Q: 1 日に何枚のカードを作成できますか?**
A: いくつでも構いませんが、量よりも質を重視してください。 50 枚の粗末なカードよりも、10 ～ 20 枚のよくできたカードの方が優れています。

**Q: どのくらいの定着率を目標にすべきですか?**
A: 推奨されるデフォルトは 90% です。レビューが多すぎる場合は 85% に調整し、重要な内容の場合は 95% に調整します。

**Q: 言語に Incrementum を使用できますか?**
A: もちろんです！語彙、文法、センテンスカードに最適です。文法パターンにはクローゼカードを使用します。

**Q: 数式はどのように処理すればよいですか?**
A: カードでは LaTeX 構文を使用してください。 OCR の場合、数学コンテンツで最良の結果を得るには、Mathpix プロバイダーを使用してください。

**Q: Anki と同期できますか?**
A：はい！ [設定] → [統合] → [Anki] で AnkiConnect を双方向同期用に設定します。

**Q: 一時停止と削除の違いは何ですか?**
A: 一時停止すると、カードが一時的に非表示になります (一時停止を解除することもできます)。削除すると完全に削除されます (バックアップから復元できます)。

**Q: どれくらいの頻度でレビューする必要がありますか?**
A: 毎日が理想的です。日数を逃した場合でも、カードは蓄積されますが、「失われる」ことはありません。可能なときに取り戻してください。

**Q: 複数のデバイスで Incrementum を使用できますか?**
A: まだ直接ではありませんが、Dropbox/Google Drive 経由でデータを同期することも、ブラウザ拡張機能を使用することもできます。

**Q: 私のデータは非公開ですか?**
A：はい！すべてのデータはローカルに保存されます。クラウド同期は暗号化されています。構成された AI プロバイダー以外のサーバーにはデータは送信されません。

**Q: カードをエクスポートするにはどうすればよいですか?**
A: 設定 → バックアップ → エクスポート、または Anki 同期を使用して .apkg 形式にエクスポートします。
