# Incrementum 用户手册

**掌握渐进阅读和间隔重复的完整指南**

---

## 简介

### 什么是增量？

Incrementum 是一款功能强大的学习应用程序，结合了两种经过验证的技术：

**增量阅读** - 随着时间的推移，以可管理的小块的形式处理大量信息。您不必从头到尾阅读文章，而是提取要点并逐渐建立理解。

**间隔重复** - 以科学优化的时间间隔复习材料，以最大程度地保留内容。 FSRS-6 和 SM-18 等算法可以预测您何时会忘记并及时安排复习。

### 关键概念

- **文档**：源材料（PDF、EPUB、文章、视频）
- **摘录**：您从文档中摘录的要点或部分
- **学习项目**：根据摘录创建的抽认卡或问答卡
- **队列**：计划审核的项目，按优先级组织
- **复习**：您积极回忆并评价您的知识的课程

## 开始使用

### 首次发布

当您首次启动 Incrementum 时，您将看到包含四个主要部分的 **仪表板**：

1. **队列** - 您的审阅队列（一开始为空）
2. **审查** - 积极的审查会议
3. **文档** - 您的文档库
4. **分析** - 进度统计

### 初始设置

1. **选择主题** - 导航至设置 → 外观 → 主题
   - 147 个可用内置主题（26 个现代主题，121 个传统主题）
   - 尝试“Modern Dark”或“Material You”以获得现代外观

2. **配置复习设置** - 设置→学习→算法
   - **算法**：FSRS-6（推荐）、SM-18 或 SM-2
   - **所需保留**：90%（默认）- 目标是您想要记住的程度
   - **每天学习**：建议初学者学习 20-50 项

3. **设置类别** - 设置 → 类别
   - 为不同主题创建类别（例如“编程”、“科学”、“语言”）
   - 类别有助于组织和过滤您的学习材料

### 你的第一个文档

让我们导入您的第一个文档：

1. 单击侧栏中的**文档**
2. 单击“**导入**”按钮（右上角）
3. 选择您的导入方式：
   - **本地文件**：选择 PDF、EPUB 或文本文件
   - **URL**：粘贴任何网址
   - **Arxiv**：粘贴研究论文 ID 或 URL
4.等待处理
   - 如果在“设置”中启用自动分割，则导入后文档将自动分割为摘录

## 文档管理

### 导入格式

|格式|描述 |使用案例|
|--------|-------------|----------|
| **PDF** |便携式文档格式|研究论文、电子书、文档 |
| **EPUB** |电子出版物|带有可重排文本的书籍、文章 |
| **降价** | `.md` 文件 |技术文档、注释 |
| **HTML** |网页 |文章、博客文章 |
| **Anki (.apkg)** | Anki 套牌包 |从 Anki 迁移 |
| **超级备忘录** |邮政编码出口 |从 SuperMemo 迁移 |
| **JSON (.json)** |抽认卡卡片组文件 |导入带有调度数据的牌组 |
| **网址** |任何网页链接 |在线文章、博客 |
| **Arxiv** |学术论文|研究文献|
| **截图** |屏幕截图 |从任何应用程序快速捕获|

### 导入文档

#### 方法一：本地文件

1. 点击**文档** → **导入**
2. 选择**本地文件**
3. 导航到您的文件并选择它
4. 增量将：
   - 提取文本内容
   - 计算阅读时间和字数
   - 提取元数据（标题、作者等）
   - 如果启用自动分段（设置 → 文档 → 导入时自动处理），自动将文档拆分为摘录

#### 方法2：URL导入

1. 复制任意网址
2. 单击**文档** → **导入** → **URL**
3. 粘贴网址
4. 单击**导入**
5. Incrementum获取并处理内容

**支持的网站：**
- 新闻文章（大多数主要网站）
- 博客文章
- 文档页面
- 介质、子堆栈等

#### 方法 3：Arxiv 论文

1. 找到一篇 Arxiv 论文（例如，`https://arxiv.org/abs/2301.07041`）
2. 复制 URL 或论文 ID (`2301.07041`)
3. 单击**文档** → **导入** → **Arxiv**
4. 粘贴 URL 或 ID
5.增量下载：
   - 完整的PDF
   - 摘要
   - 作者
   - 出版日期
   - 参考文献

#### 方法 4：JSON Deck 导入

从 JSON 文件导入抽认卡组，其中包括日程安排数据（间隔、轻松因素、复习历史记录）。

**通过文件选择器导入：**

1. 点击**文档** → **导入** → **JSON**
2. 选择您的“.json”甲板文件
3. Incrementum 创建一个牌组文档并导入所有卡牌，保留：
   - 日程安排（间隔、轻松因素、截止日期）
   - 回顾历史（重复、失误、保留率）
   - 卡状态（新卡、正在审核或已暂停）

**通过拖放导入：**

将“.json”文件直接拖到应用程序窗口上。如果文件与预期的牌组格式匹配，则会自动导入。

**JSON 甲板格式：**

该文件应该是将问题文本映射到卡片数据的平面对象：

```json
{
  “细胞的动力源是什么？”：{
    "answer": "线粒体。",
    “主题”：“生物学”，
    "deck_name": "细胞生物学",
    “轻松系数”：2.6，
    “间隔天数”：7，
    “重复次数”：3，
    “due_at”：“2026-04-20T12:00:00Z”
  }
}
````

**注释：**
- 每个“.json”文件都会创建一副牌。卡组名称来自“deck_name”字段。
- 进口卡默认使用SM-2算法。您可以在导入后切换算法。
- 两次删除同一文件不会创建重复项 - 现有的卡将被跳过。
- 标记为“known_pile: true”的卡将作为暂停导入。

### 文档查看器

导入后，打开任何文档即可访问：

**查看器功能：**
- **页面导航**：滚动页面/部分
- **缩放**：调整文字大小
- **全屏**：无干扰阅读
- **搜索**：在文档中查找文本
- **目录**：跳转到各个部分（如果有）**注释工具：**
1. **突出显示文本**：选择文本→选择突出显示颜色
   - 黄色：重要概念
   - 绿色：示例
   - 蓝色：定义
   - 红色：关键点
   - 紫色：相关主题

2. **创建提取**：选择文本→单击“提取”按钮
   - 提取出现在提取选项卡中
   - 稍后可以转换为抽认卡

3. **添加注释**：选择文字→点击“注释”按钮
   - 附上您的想法/笔记
   - 注释与摘录一起出现

### 文档组织

**类别：**
- 将每个文档分配到一个类别
- 按类别过滤文档
- 类别继承到摘录和卡片

**搜索：**

- 按标题、日期、字数排序

## 学习系统

### 了解 FSRS-6

**FSRS-6**（自由间隔重复调度程序）是一种现代算法，它：

1. **跟踪内存状态**：对每张卡的内存强度进行建模
2. **预测遗忘**：估计您何时会忘记每件物品
3. **优化安排**：在最佳时间安排审核
4. **适应您**：从您的表现模式中学习

**关键指标：**
- **稳定性**：记忆能持续多久（越高=越稳定）
- **难度**：该项目对您来说有多难（1-10 级）
- **可检索性**：当前召回概率 (0-100%)

### 评级系统

在审核期间，根据您的回忆对每个项目进行评分：

|评级 |标签|描述 |典型间隔 |
|--------|--------|-------------|--------------------|
| **1** |再次 |完全停电|约 10 分钟 |
| **2** |硬|付出巨大努力才记住| 1-2 天 |
| **3** |好 |想起来了| 5-7 天 |
| **4** |简单|回忆起来毫不费力| 10-14 天 |

**预览间隔：**
在评分之前，Incrementum 会准确显示所有四个评分选项中每张卡接下来出现的时间。用它来优化您的日程安排！

### 卡牌类型

#### 1. 基本抽认卡
简单的正面/背面卡片

**前：** 法国的首都是哪里？
**返回：** 巴黎

**最适合：** 事实、定义、词汇

#### 2. 完形填空删除
填空式

**文本：** {{France}}的首都是巴黎。

**显示为：** _____ 的首都是巴黎。

**最适合：** 情境学习、人际关系

#### 3. 问答卡
问题和答案对

**问：**解释一下 TCP 和 UDP 之间的区别。
**A:** TCP 是面向连接的，有保证的传送； UDP 是无连接的，没有保证。

**最适合：** 概念、解释

#### 4. 图像遮挡
隐藏图像的一部分（图表）

**最适合：** 解剖学、地图、图表

### 创建卡片

####摘录

1.阅读时，选择重要的文字
2. 单击“**提取**”创建提取
3. 在 **摘录** 选项卡中，检查您的摘录
4. 在任意摘录上单击“**创建卡片**”
5. 选择卡片类型（抽认卡、完形填空、问答）
6.编辑卡片内容
7. 单击“**保存**”

该卡现已安排审核！

#### 手动创建

1. 点击 **队列** → **添加项目**
2. 选择卡类型
3. 输入正面/背面内容
4. 选择类别
5. 单击“**创建**”

#### 人工智能发电

如果您配置了 AI：

1. 选择摘录或文档部分
2.点击**生成卡片**
3. AI会自动创建多张卡片
4. 根据需要进行审查和编辑
5. 保存最好的

## 牌组经理

**甲板管理器**是一个全屏视图，用于浏览、检查和编辑您的抽认卡甲板及其卡片。从 Review 主页上的 **Deck Manager** 按钮将其打开。

### 浏览套牌

- 左侧边栏列出了您的所有牌组以及牌数和今日到期指示器。
- 单击一副牌组将其展开 - 一次仅展开一副牌组。
- 每个牌组的标签过滤器在牌组名称下方显示为小药丸。

### 卡牌列表

当牌组展开时，其牌将出现在虚拟的可滚动列表中。每个卡行显示：

- **州徽章** — 颜色编码：蓝色（新）、橙色（学习）、绿色（复习）、红色（重新学习）
- **问题预览** — 最多 80 个字符
- **截止日期** — 相关标签（今天、明天、5天、逾期）
- **难度** — 1–10 迷你进度条
- **间隔** — 当前审核间隔（以天为单位）
- **审核次数** — 该卡已被审核了多少次
- **水蛭指示器** — 黄色警告图标，表示失败次数超过 5 次的卡片

### 排序和过滤

**按截止日期、状态、难度、间隔、复习次数或失误对卡片进行排序。再次单击排序按钮可切换升序/降序。

**过滤**：
- **状态** — 新的、学习的、回顾的、重新学习的
- **到期状态** — 今天到期、逾期、未到期

使用顶部的搜索栏按问题文本或标签名称进行**搜索**。

### 内联卡片编辑器

单击任何卡行以展开其下方的内联编辑器：

- 直接编辑**问题**、**答案**和**标签** - 无需模式。
- **完形填空卡**显示完形填空文本以及突出显示的删除范围。
- **复杂的卡片类型**（多项选择、图像遮挡）显示带有“在 Studio 中编辑”链接的只读预览。
- **挂起/取消挂起**只需单击一下即可切换。
- 更改通过**乐观更新**保存 - UI 立即更新并在保存失败时回滚。

### 牌组统计面板

右侧边栏显示扩展甲板的统计信息：

- **今天到期** — 现在到期的卡数
- **保留率** — 根据失效率估算
- **平均难度** — 牌组中所有卡牌的难度
- **水蛭计数** — 失误次数超过 5 次的卡片（单击以仅过滤水蛭）
- **成熟度细分** — 显示新/学习/年轻/成熟卡分布的堆叠条
- **7 天预测** — 显示下周预计到期计数的迷你图
- **FSRS 内存健康状况** — 平均稳定性和难度，带有颜色编码的健康状况指示器

### 批量操作

使用复选框选择多张卡，然后使用批量操作工具栏：

- **挂起/取消挂起** — 批量切换挂起
- **删除** — 删除选定的卡（需确认）
- **重新标记** — 一次在所有选定的卡片上添加或删除标签

### 键盘快捷键

|关键|行动|
|-----|--------|
| '逃脱' |折叠内联编辑器，或折叠扩展面板 |

## 审核流程

### 开始审查会议

1. 点击侧边栏中的**查看**
2. 查看今天（以及即将到期）到期的卡片
3. 单击“**开始审核**”开始

### 评论界面

**卡片展示：**
- 显示卡片正面（问题或提示）
- 按 **空格** 或单击以显示答案
- 答案出现在下面

**混合审查会议（卡片+文档）：**
- 复习课程可以包括需要阅读的**学习项目**和**文档**。
- 当文档出现时，您可以直接从会话卡打开它。
- 对文档进行评级会安排其下一次阅读日期，就像卡片安排其下一次审阅一样。

**评分界面：**
显示答案后，会出现四个评分按钮：

````
[再次][难][好][简单]
  ～10m ～2d ～7d ～14d
````

每个按钮都会显示**下一次审核日期** - 这是**预览间隔**功能！

**恢复操作（查看队列检查器）：**
当学习项目的日程需要快速调整时，请使用这些：

- **压缩间隔**：拉近下一次审核的时间（更短的间隔）。
- **智能重新安排**：将项目移至“立即到期”。
- **降级频率**：推迟下一次审核（更长的间隔）。

这些操作仅适用于**学习项目**并立即更新时间表。

### 键盘快捷键（审阅模式）

|关键|行动|
|-----|--------|
| '空间' |显示答案 |
| `1` |评价“再次” |
| `2` |评价“困难”|
| `3` |评价“良好”（推荐默认值）|
| `4` |评价“简单”|
| `Ctrl+Enter` |显示答案 |
| `Ctrl+1/2/3/4` |评分但不显示答案 |
| `Esc` |暂停/结束会话 |
| `Ctrl+E` |编辑当前卡（尚未实施）|
| `Ctrl+D` |删除当前卡（也全局用于“转到仪表板”）|

### 会话管理

**回顾会议特点：**
- **进度条**：显示剩余的卡片
- **时间跟踪**：显示会话持续时间
- **休息计时器**：可选地每 N 张牌就休息一次
- **会话限制**：设置每次会话的最大卡数或时间

**结束会话：**
- 完成后点击**完成**
- 或设置限制（设置→查看→会话限制）
- 未完成的卡片仍需在下一次会议中使用

### 复习策略

#### 日常回顾

1. **上午课程**（15-30 分钟）
   - 审查隔夜到期的卡
   - 专注于较难的项目

2. **晚间课程**（15-30 分钟）
   - 白天添加的复习卡
   - 通过阅读创建新卡片

#### 管理待办事项

如果您有很多到期卡（>100）：

1. **关注新卡**：限制审核次数为 20-30 次/天
2. **使用过滤器**：按类别审查（不要不知所措）
3. **补习班**：周末补习班
4. **调整保留率**：暂时降低至85%（评论较少）

#### 处理“再次”牌

评级为“再次”的卡片会快速重新出现（10 分钟）。策略：

- **立即重新学习**：在同一会话中再次复习卡片
- **单独会议**：当天晚些时候再次复习卡片
- **理解问题**：如果重复多次，卡片可能写得不好

## 队列管理

### 了解队列

**队列**包含计划审查的所有项目，组织者：

- **到期日期**：到期较早的项目首先出现
- **优先级**：用户设置优先级（0-100）
- **类别**：主题领域
- **卡片类型**：抽认卡、完形填空等。

### 队列视图

#### 正当观点
显示今天到期和逾期的项目，按到期时间排序

#### 预定视图
显示所有预定项目，包括未来的评论

#### 新视图
显示尚未审核的新创建的卡片

### 队列操作

**过滤：**
- 按类别（例如“仅显示节目”）
- 按卡片类型（例如，“仅显示完形填空卡片”）
- 按优先级范围（例如，“显示优先级 80+”）

**排序：**
- 截止日期（默认）
- 优先权
- 难度
- 随机（多样化）

**批量操作：**
1. 选择多个项目（复选框）
2. 选择操作：
   - **更改类别**：移至不同类别
   - **设置优先级**：批量更新优先级
   - **暂停**：暂时隐藏评论
   - **删除**：永久删除

### 优先系统

对任何项目设置优先级 0-100：

- **100**：关键（必须学习）
- **80-90**：重要
- **60-70**：正常优先级
- **40-50**：低优先级
- **0-20**：存档/参考

**优先调度：**
优先级较高的项目在混合评论中显示的频率更高。

### 智能队列

使用过滤器创建自定义队列：

**队列示例：**

### 学习设置

#### 算法选择

Incrementum支持三种调度算法。选择最适合您的学习风格的一种：

**FSRS-6（推荐）：**
- 现代、研究支持
- 适应个人记忆
- 预测遗忘时间
- 更少的评论，更好的保留

**SM-18（SuperMemo 18）：**
- 最新的 SuperMemo 算法，对原始应用程序进行逆向工程
- 使用 3D SInc（稳定性增加）查找矩阵来衡量难度、稳定性和可检索性
- 通过跟踪平均更新来明确难度跟踪
- 指数遗忘曲线模型：`R = 0.9^(t/S)`
- 复杂的故障处理与失效相关的稳定性降低

**SM-2（经典）：**
- 传统 SuperMemo 2 算法（公开记录）
- 更简单、可预测
- 需要更多评论

#### 参数

**所需保留：** 0.70 - 0.95
- **90%**（默认）：平衡保留和审核负载
- **85%**：评论较少，留存率稍低
- **95%**：最大保留率，更多评论

**每天学习：** 10 - 100
- **20**（默认）：大多数用户均可管理
- **50**：用于强化学习阶段
- **10**：审查负担较轻

**每天回顾：** 50 - 500
- **200**（默认）：合理的每日限额
- **500**：用于清理积压订单
- **50**：轻度审查日

#### 间隔设置

**新卡间隔：**
- 毕业间隔（良好评级）：1-10天
- 简单间隔：3-21天
- 最短间隔：1天

**最大间隔：**
- 最长间隔上限（默认 365 天）
- 防止卡片安排得太远

**长型安全帽（视频/文章）：**
- 对于长视频/文章，正面评级（“好”/“简单”）是覆盖范围感知的。
- 如果您花费的时间少于预计内容时间的 **25%**，则下一个时间间隔的上限为 **1 天**。
- 如果您的支出低于 **50%**，则下一个间隔的上限为 **2 天**。
- 如果您的支出低于 **75%**，则下一个间隔的上限为 **4 天**。
- 这可以防止长篇内容在部分进度之后被安排得太晚。
- 应用时，调度程序原因包括**持续时间感知上限**注释以提高透明度。

#### 备份与恢复

Incrementum 提供完整的备份和恢复系统来保护您的学习数据并在设备之间迁移。

#### 完整的应用程序备份

**备份什么：**
- **设置**：所有首选项、主题、学习参数
- **文档**：所有带有元数据的导入文档
- **摘录**：所有亮点和摘录内容
- **学习项目**：所有抽认卡、完形填空、问答卡
- **调度数据**：算法记忆状态（稳定性、难度、间隔）、截止日期
- **集合**：所有集合和文档分配
- **UI状态**：侧边栏状态，主题首选项
- **可选**：实际文档文件（PDF、EPUB 等）

**创建备份：**

1. 转到 **设置 → 导入/导出 → 完整应用程序备份**
2. 单击“**打开备份和还原**”
3. 选择**导出备份**
4. 添加可选标签（例如“重新格式化 PC 之前”）
5. 选择是否包含文档文件：
   - **仅元数据**：较小的文件 (~KB-MB)，单独重新导入文件
   - **包含文件**：更大的文件（~MB-GB），完整的独立备份
6. 单击 **导出备份** 并保存 `.incrementum` 文件

**文件格式：**
- 扩展名：`.incrementum`
- 格式：带标题注释的 JSON
- 命名：`incrementum-backup-[标签]-[日期]-[时间].incrementum`

**从备份恢复：**

1. 转到 **设置 → 导入/导出 → 完整应用程序备份**
2. 单击“**打开备份和还原**”
3. 选择**导入备份**
4. 选择您的“.incrementum”文件
5. 预览备份中的内容：
   - 文件计数
   - 提取计数
   - 学习项目计数
   - 收藏数量
   - 是否包含文件
6. 配置导入选项（可选）：
   - **导入内容**：选择特定数据类型
   - **重复处理**：跳过、替换或合并
   - **导入文件**：是否恢复文档文件
7. 单击“**导入备份**”
8.等待导入完成（显示进度）

**重复处理策略：**
- **跳过**：跳过已经存在的项目（大多数情况下推荐）
- **替换**：用备份版本覆盖现有项目
- **合并**：创建所有项目的新副本（可能会创建重复项）

**使用案例：**|场景|推荐方法 |
|----------|---------------------|
| **迁移到新计算机** |使用文件导出，在新机器上导入 |
| **重大更改前的备份** |快速仅元数据备份 |
| **设备之间同步** |导出/导入工作流程 |
| **分享收藏** |导出特定集合 |
| **归档旧数据** |出口并长期储存|
| **重新格式化后恢复** |导入带有文件的完整备份 |

**重要说明：**
- **调度保留**：所有算法类型的所有调度数据（稳定性、难度、到期日期）都被准确保留
- **文件路径**：在没有文件的情况下导入时，您需要重新导入原始文档。 Incrementum将通过内容哈希匹配它们并恢复元数据
- **版本兼容性**：备份向前兼容，但可能不适用于较旧的应用程序版本
- **存储**：确保备份安全 - 它们包含您的个人学习数据

### OCR（光学字符识别）

从图像中提取文本：

**支持的提供商：**
- GLM-OCR（本地）——通过 llama.cpp 或 vLLM 进行多模式 OCR
- 超正方体（本地）
- 谷歌云视觉
-AWS Textract
- Azure 计算机视觉
- 标记（本地）— PDF 到 Markdown
- Nougat（本地）——带有数学的科学文档

**使用案例：**
- 截图捕捉
- 扫描文件
- 带文字的图像
- 手写笔记

**设置（云提供商）：**
1. 设置→OCR
2. 选择提供商（Google、AWS 或 Azure）
3. 配置 API 密钥和凭据
4. 选择语言
5. 使用示例图像进行测试

**设置（带有 llama.cpp 的 GLM-OCR）：**

llama.cpp 为 GLM-OCR 提供轻量级本地 LLM 服务器，无需 GPU。它在端口 8080 上使用兼容 OpenAI 的 API。

1. **构建 llama.cpp** （如果尚未构建）：
   ````bash
   git 克隆 https://github.com/ggml-org/llama.cpp.git
   cd 骆驼.cpp
   cmake -B 构建
   cmake --build 构建 --config Release -j$(nproc)
   ````

2. **下载多模式模型**（GGUF格式）：
   ````bash
   # Qwen2.5-VL（推荐用于OCR）
   Huggingface-cli 下载 bartowski/Qwen2.5-VL-7B-Instruct-GGUF \
     Qwen2.5-VL-7B-Instruct-Q4_K_M.gguf --local-dir 模型/
   ````

3. **启动服务器**：
   ````bash
   ./build/bin/llama-server \
     -m 型号/Qwen2.5-VL-7B-Instruct-Q4_K_M.gguf \
     --端口 8080 --主机 0.0.0.0 -c 16384 -t $(nproc)
   ````

4. **增量配置**：
   - 设置 → OCR → 提供商：**GLM-OCR（本地）**
   - 后端：**vLLM (GPU)**（这是 llama.cpp/vLLM 模式 — 适用于两者）
   - 端点：`http://localhost:8080/v1`
   - 模型：您的模型文件名（例如“Qwen2.5-VL-7B-Instruct-Q4_K_M.gguf”）**性能提示：**
- 对于长文档使用 `-c 16384` 或更高值（默认 4096 对于大多数 OCR 任务来说太小）
- 使用 `-t $(nproc)` 使用所有 CPU 线程
- Q4_K_M 量化为 CPU 推理提供最佳质量/速度权衡
- 对于 GPU 加速，构建具有 CUDA、Metal 或 Vulkan 支持的 llama.cpp

**设置（带有 vLLM 的 GLM-OCR）：**

vLLM 为大型模型提供 GPU 加速推理。需要具有足够 VRAM 的 NVIDIA GPU。

````bash
pip安装-U vllm
vllm 服务 zai-org/GLM-OCR --allowed-local-media-path / --port 8080
````

然后以相同的方式配置 Incrementum（端点“http://localhost:8080/v1”）。

**设置（GLM-OCR 与 Ollama）：**

最简单的入门选项 — Ollama 自动管理模型下载和运行时。

1. 设置 → OCR → 提供商：**GLM-OCR（本地）**
2. 后端： **Ollama (CPU)**
3. 单击**下载 Ollama**（如果未安装）
4. 单击“**启动运行时**”
5. 设置模型（例如 `llava:7b` 或 `qwen2-vl:7b`）
6. 单击**拉模型**

**数学 OCR：**
- 方程的专门处理
- 乳胶输出
- 符号识别
- 最适合：科学论文、教科书

## 提示和最佳实践

### 卡片制作

**做：**
- 使卡片具体化（每张卡片一个事实）
- 使用简单、清晰的语言
- 在答案中包含上下文
- 添加相关示例
- 使用完形填空来建立关系
- 保持问题简洁

**不要：**
- 将多个事实放入一张卡中
- 使用含糊的措辞
- 让问题太简单或太难
- 复制大文本块
- 使用没有定义的缩写

**示例 - 坏卡：**
````
问：线粒体的功能是什么？
它与细胞呼吸中 ATP 的产生有关吗？
答：【段落解释】
````

**示例 - 好卡：**
````
卡1：
问：线粒体的主要功能是什么？
A：通过细胞呼吸产生ATP

卡2：
问：线粒体利用什么过程产生 ATP？
A：细胞呼吸（有氧）

卡3：
问：线粒体产生的能量货币是什么？
A：ATP（三磷酸腺苷）
````

### 学习常规

**每日时间表（20-30 分钟）：**
1. **上午**：查看到期卡（15 分钟）
2. **全天**：从阅读中创建摘录
3. **晚上**：根据摘录创建卡片（10-15 分钟）

**每周时间表：**
- **周一至周五**：定期审查和卡片创建
- **周六**：较长的学习课程（1-2 小时）
- **周日**：回顾分析、调整目标、组织

**管理大容量：**
- 设置每日审核限制（例如 50 张卡片）
- 按类别划分优先级（专注于一个主题）
- 使用智能队列来分解任务
- 每20-30分钟休息一次

### 保留优化

**提高保留率：**
- 诚实评分（不要夸大评分）
- 持续回顾（每天最好）
- 充足的睡眠（睡眠期间记忆会巩固）
- 主动回忆（不要偷看，先思考）
- 间隔评论（不要死记硬背）

**处理遗忘：**
- 正常遗忘 10-20%（取决于目标保留率）
- “再次”卡是学习机会
- 如果经常忘记（>30%），请考虑：
  - 降低所需的保留率 (85-90%)
  - 创建更简单的卡片
  - 添加更多上下文
  - 更频繁地复习

### 类别组织

**最佳实践：**
- 从广泛开始，然后细分
- 示例：`编程`→`编程/Python`→`编程/Python/异步`
- 使用一致的命名
- 不要创建太多（5-10 个是可以管理的）
- 合并未使用的类别

**类别结构示例：**
````
├── 编程
│ ├── Python
│ ├── 铁锈
│ └── 算法
├── 语言
│ ├── 西班牙语
│ └── 日语
├── 科学
│ ├── 物理
│ └── 生物学
└── 专业
    ├── 项目管理
    └── 系统设计
````

### 优先级管理

**优先准则：**
- **100（关键）**：考试准备、紧急工作项目
- **80-90（高）**：当前课程，主动学习
- **60-70（中）**：持续的兴趣、常识
- **40-50（低）**：很高兴知道，补充
- **0-20（存档）**：仅供参考，很少回顾

**优先调度：**
- 专注于每日评论的 80+ 优先级
- 每隔几天复习60-70
- 每周回顾 40-50 次
- 每月或按需审核 0-20 次

### 使用预览间隔

**预览间隔**功能可以准确显示所有四个评级的每张卡接下来出现的时间。

**使用方法：**
1.读卡
2.检查评级按钮下方的预览间隔
3. 根据以下条件选择评级：
   - 您当前的回忆
   - 您想多久再次看到它
   - 您的日程安排（例如即将举行的考试）**策略示例：**
- 两周内考试：对重要卡片进行评分“简单”，以便很快再次看到它们
- 忙碌的一天：评价“好”或“简单”以间隔评论
- 想要掌握：评价“难”以更频繁地复习

### 管理不堪重负

**到期卡太多？**
1. 设置审核限额（设置→审核→每日上限）
2. 关注高优先级项目
3. 暂时暂停低优先级类别
4.考虑稍微降低所需的保留率

**内容太多而无法处理？**
1. 逐步导入文件
2.只提取关键点（不是全部）
3.有选择地创建卡片
4.使用类别来组织

**倦怠？**
1.休息一下（没关系！）
2.减少每日限额
3. 暂停非关键类别
4.一次专注于一个类别

## 故障排除

### 常见问题

#### 卡片未出现在审核中

**可能的原因：**
- 今天审核的所有卡片
- 卡被暂停
- 过滤活跃的隐藏卡

**解决方案：**
1. 在“审阅”选项卡中检查“到期”计数
2. 检查队列 → 确保卡未暂停
3. 清除过滤器
4. 检查审核日期（也许是将来安排的卡片）

#### 保留率低

**症状：** 忘记很多卡片，频繁“再次”评级

**解决方案：**
1. **检查卡片质量**：卡片是否清晰？每张卡一个事实？
2. **降低所需保留率**：尝试 85% 而不是 90%
3. **更频繁地复习**：每天复习，而不是临时抱佛脚
4. **添加上下文**：答案中的更多信息
5. **简化卡片**：将复杂的卡片分解为更简单的卡片

#### 同步冲突

**症状：** 重复卡、同步后数据不匹配

**解决方案：**
1. 选择冲突解决策略（设置→同步）
   - **本地胜利**：保留您的更改
   - **远程获胜**：接受服务器更改
   - **询问**：手动解决每个冲突
2.定期同步以尽量减少冲突
3. 使用一台主要设备

#### 导入失败

**症状：** 文档导入失败或错误

**解决方案：**
1. **检查文件格式**：确保支持的格式（PDF、EPUB等）
2. **检查文件大小**：非常大的文件可能会超时
3. **检查 URL**：某些网站阻止自动访问
4. **检查互联网**：URL导入需要连接
5. **尝试替代方案**：对网页内容使用复制粘贴

#### 性能问题

**症状：** 加载缓慢、滞后、冻结

**解决方案：**
1. **大型数据库**：存档旧卡（设置→数据→存档）
2. **许多图像**：图像加载速度变慢
3. **系统资源**：关闭其他应用程序
4. **重建数据库**：设置→数据→重建（最后的手段）

#### OCR 不工作

**症状：** OCR 失败或结果不佳

**解决方案：**
1. **检查 API 密钥**：有效且有积分（云提供商）
2. **检查图像质量**：清晰、高分辨率的图像效果最佳
3. **检查语言**：选择正确的语言
4. **尝试替代提供商**：有些对于某些内容效果更好
5. **本地 OCR**：如果互联网问题，请使用 Tesseract

#### llama.cpp 没有响应

**症状：** “调用 LLM 时出错”或连接拒绝 localhost:8080

**解决方案：**
1. **检查服务器是否正在运行**: `curl http://localhost:8080/v1/models`
2. **启动服务器**：参见上面的[OCR设置](#ocr-optical-character-recognition)
3. **上下文大小太小**：使用 `-c 16384` 或更高版本重新启动
4. **端口正在使用**：另一个进程可能正在使用端口8080；检查“lsof -i :8080”
5. **内存不足**：使用较小的量化（Q3_K_M 而不是 Q4_K_M）或较小的模型

#### 奥拉马未开始

**症状：** GLM-OCR Ollama 运行时无法启动

**解决方案：**
1. **安装 Ollama**：使用“设置”→“OCR”中的“下载”按钮，或从 ollama.com 安装
2. **检查二进制路径**：如果不在默认位置，则设置 Ollama 二进制路径
3. **Linux权限**：您可能需要`sudo`来安装或运行Ollama服务

## 术语表

**提取**：从文档中提取的一段内容，潜在的卡片材料

**学习项目**：任何要学习的项目（抽认卡、完型填空、问答等）

**队列**：安排审核的所有项目，按优先级组织

**回顾会议**：积极回忆和评分卡片的一段时间

**FSRS**：Free Spaced Repetition Scheduler，现代算法优化审阅时间（FSRS-6 是当前版本）

**间隔**：审核之间的时间（例如 7 天）

**稳定性**：内存持续多长时间（FSRS 指标）

**难度**：某个项目对您来说有多难，1-10 级（FSRS 指标）

**可检索性**：当前召回概率，0-100%（FSRS 指标）

**所需保留率**：目标保留率（通常为 90%）

**预览间隔**：显示每个评级选项的下一次审核日期的功能

**完形填空**：填空卡类型

**暂停**：暂时隐藏评论中的项目

**类别**：组织的主题领域

**标签**：跨类别组织的自定义标签

**优先级**：用户设置的重要性（0-100）

## 常见问题解答

**问：如何将新闻通讯添加到 Incrementum？**
答：您可以通过两种方式添加新闻通讯：
1. **新闻通讯目录**：点击 RSS → 新闻通讯图标 (📬) → 浏览并订阅精选新闻通讯
2. **直接 URL**：复制任何时事通讯 URL（Substack、Beehiiv 等）→ RSS → 添加 Feed → 粘贴 URL。 Incrementum 将自动发现 RSS 提要。

**问：支持哪些新闻通讯平台？**
答：Incrementum 支持来自 Substack、Beehiiv、Ghost blogs、Buttondown、ConvertKit、Revue、Medium 和 WordPress 网站的 RSS 源。大多数时事通讯都会发布 RSS 源 - 检查时事通讯网站上的 RSS 链接或尝试将“/feed”添加到 URL。

**问：我每天应该复习多少张卡片？**
答：从每天 20-50 个开始。根据您的日程安排和目标进行调整。一致性比数量更重要。

**问：我每天可以创建多少张卡片？**
答：想要多少就多少，但注重质量而不是数量。 10-20 张制作精良的卡片比 50 张劣质卡片要好。

**问：我的目标留存率应该是多少？**
答：90% 是建议的默认值。如果您的评论过多，则调整为 85%；对于关键材料，则调整为 95%。

**问：我可以对语言使用 Incrementum 吗？**
答：当然！它非常适合词汇、语法和句子卡。使用完形填空卡来了解语法模式。

**问：如何处理数学方程？**
答：在卡片中使用 LaTeX 语法。对于 OCR，请使用 Mathpix 提供程序以获得数学内容的最佳结果。

**问：我可以与 Anki 同步吗？**
答：是的！在设置 → 集成 → Anki 中配置 AnkiConnect 进行双向同步。

**问：暂停和删除有什么区别？**
A：暂停会暂时隐藏卡牌（可以取消暂停）。删除将永久删除（可以从备份中恢复）。

**问：我应该多久复习一次？**
答：每天是理想的选择。如果你错过了几天，卡片会累积，但不会“丢失”——只要有机会就补上。

**问：我可以在多个设备上使用 Incrementum 吗？**
答：还不能直接同步，但您可以通过 Dropbox/Google Drive 同步数据，或使用浏览器扩展程序。

**问：我的数据是私密的吗？**
答：是的！所有数据都存储在本地。云同步已加密。除配置的 AI 提供程序外，不会向服务器发送任何数据。

**问：如何导出我的卡？**
A：设置 → 备份 → 导出，或使用 Anki 同步导出为 .apkg 格式。
